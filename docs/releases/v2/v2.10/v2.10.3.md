This is a new patch release of the `alex-c-line` package. It includes small, non-breaking changes and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Make the allowed licenses more permissive.
  - The following licenses have been added to the allowed licenses:
    - `BlueOak-1.0.0`
    - `Python-2.0`
    - `0BSD`
    - `Unlicense`
    - `CC0-1.0`
  - These have been added alongside:
    - `MIT`
    - `MIT-0`
    - `ISC`
    - `Apache-2.0`
    - `BSD-2-Clause`
    - `BSD-3-Clause`
  - All of the above licenses have been deemed permissible enough such that using them in our projects is unlikely to cause any legal issues.
- Prevent SPDX expressions (e.g. `(MIT OR ISC)`) from surfacing as a warning if the allow list allows one of the chosen licenses.
  - If one or more of the licenses are permitted by the allow list, then the expression is to be treated as valid.

## Additional Notes

- Eventually I will make the allow list configurable so that individual projects can specify their own allowed licenses.
- More information on what that may look like will be coming soon.
