## Description of Changes

<!-- user-editable-section-start -->
- Add a rule to enforce exact version pins in `pyproject.toml` dependencies.
  - This enforces that Python projects keep in-line with my current dependency policy that direct dependencies should be exactly pinned, and those pins should be updated in a controlled way every week.
  - By introducing this rule, it ensures that all Python dependencies are pinned exactly (at `==`), so even if the exact save default I tend to have ends up failing, the tooling will still catch attempts to bring in non-exact pins and error against them.
