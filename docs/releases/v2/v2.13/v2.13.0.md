This is a new minor release of the `alex-c-line` package. It introduces new features and other backwards-compatible changes that should require little to no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Add a `--status` flag to `template release-note create`
  - This allows users to set the status of the release note at creation time.
- Add an `--update-version` flag to `template release-note create`
  - This updates the `package.json` version alongside creating the release note, which makes it easier to use to prepare releases as well as creating the initial release note.

## Additional Notes

- In the case of my workflows, both of the above means that we now have a way to skip over `commit-version-change` having to create a second pull request to do the release preparation and can instead prepare the release from the feature pull request itself.
- This is a lot nicer so that quick and urgent fixes can go live as soon as the initial pull request is done, whereas larger major releases still have a way to control the release a little more using the old `commit-version-change` approach.
