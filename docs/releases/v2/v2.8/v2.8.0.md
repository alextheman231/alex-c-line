# v2.8.0 (Minor Release)

**Status**: In progress

<!-- alex-c-line-release-summary-start -->
This is a new minor release of the `alex-c-line` package. It introduces new features and other backwards-compatible changes that should require little to no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Add a `--content` option to `alex-c-line template release-note create`
  - This will allow the release note to start with some initial content as provided in the CLI.
  - This is expected to help with automated releases, where we may want to allow a GitHub Action to create a new release note, for example.

## Additional Notes

- This may help in addressing the issue brought up in the [Additional Notes of release note v2.7.3](https://github.com/alextheman231/alex-c-line/blob/main/docs/releases/v2/v2.7/v2.7.3.md).
- The plan now is to have a workflow that tries to create a patch release on the first of every month at 9pm.
- This will ensure that the currently published package will never end up being more than a month out-of-sync, even if work on the package freezes for some time. In particular, it ensures that dependency updates merged into the repository will always be reflected in the latest published package within at most one month.
