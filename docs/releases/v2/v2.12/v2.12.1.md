## Description of Changes

<!-- user-editable-section-start -->
- Cache the birthday notifications by version and birthday ID rather than just birthday ID.
  - Basically what this means is that, whenever you update the CLI you will get the birthday message once again.
  - I think this is acceptable - I'm unlikely to release that many updates on my birthday so there shouldn't be too much spam.
  - But in fairness, I'm also not necessarily the type of person **not** to work on an `alex-c-line` feature on my birthday either. After all, what better birthday present than a better developer experience for myself in other projects?
