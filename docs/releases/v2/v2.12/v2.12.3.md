## Description of Changes

<!-- user-editable-section-start -->
- Use `Temporal` for the birthday notifications.
  - I decided to migrate those because they're relatively low-risk.
  - Any errors would end up getting swallowed by the try/catch, and there is also an explicit undefined guard, so the worst case is just that you won't get to see the CLI celebrate my birthday or its birthday.
  - This is an acceptable trade-off in my opinion. It would make me sad that you can't celebrate my birthday because you're not on the latest Node version, but it at least won't stop you from using the CLI!
