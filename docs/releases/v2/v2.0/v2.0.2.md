This is a new patch release of the `alex-c-line` package. It fixes issues with the package and should require no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Fix an issue with `reject: false` not working correctly with pre-commit step runner.
    - The step runner would previously allow this as an option, however it did not affect the step runner previously.
    - This is because the step runner itself sets a default option of `{reject: false}` so that it can override the default ExecaError with a nicer program error.
    - The logic for this did not account for the potential binding option also setting `reject: false`, hence it got ignored.
    - This change ensures that when the step is executed, the runner respects the `reject` option and skips throwing the error when it is set to `false`.

## Additional Notes

- While we're here, here's a recommended directory setup for those who would like to use the TypeScript configs.
- To do this, I would first recommend creating a `configs/alex-c-line` folder in your repository. It may look something like this:

```
configs
└── alex-c-line
    ├── eslint.config.js
    ├── index.ts
    ├── preCommit.ts
    └── pullRequestTemplate.ts
```

Example `configs/alex-c-line/index.ts` file:

```typescript
import type { AlexCLineConfig } from "alex-c-line/configs";

import preCommitConfig from "configs/alex-c-line/preCommit";
import pullRequestTemplateConfig from "configs/alex-c-line/pullRequestTemplate";

import { scripts } from "package.json" with { type: "json" };

const alexCLineConfig: AlexCLineConfig<keyof typeof scripts> = {
  preCommit: preCommitConfig,
  template: {
    pullRequest: pullRequestTemplateConfig,
  },
};

export default alexCLineConfig;
```

Example `configs/alex-c-line/preCommit.ts` file:

```typescript
import type { PreCommitConfig } from "alex-c-line/configs";

import { scripts } from "package.json" with { type: "json" };

const preCommitConfig: PreCommitConfig<keyof typeof scripts> = {
  packageManager: "pnpm",
  steps: [
    "build",
    "format",
    "lint",
    "test",
    async (stepRunner) => {
      const { exitCode } = await stepRunner({
        reject: false,
      })`git diff --cached --quiet tests/end-to-end tsdown.config.ts`;
      // If these files did not change, don't run end-to-end
      if (exitCode === 0) {
        return;
      }
      await stepRunner`pnpm run test-end-to-end`;
    },
  ],
};

export default preCommitConfig;
```


- After that, using a build tool of your choice, build the `configs/alex-c-line/index.ts` file. It doesn't matter where you build it to necessarily, but keep it out of the publishable dist folder because otherwise it will be included in your own project's build.
    - **TIP:** I would recommend exporting it to `.alex-c-line/configs` in case I eventually add direct support in `alex-c-line` for building configs. That way, the syntax should be relatively similar if I ever do add this direct support.

- Example config in tsdown:

```typescript
import { defineConfig } from "tsdown";

export default defineConfig([
  {
    entry: ["src/root/index.ts"],
    format: ["esm", "cjs"],
    dts: true,
    clean: true,
    fixedExtension: false,
  },
  {
    entry: ["configs/alex-c-line/index.ts"],
    outDir: ".alex-c-line/config",
    format: ["esm"],
    dts: true,
    clean: true,
    fixedExtension: false,
  },
]);

```

- In your `alex-c-line.config.js` file, reference the built output like so:

```typescript
import alexCLineConfig from "./.alex-c-line/config/index.js";

export default alexCLineConfig;
```

- You should now be able to benefit from all of TypeScript's nice typing features, including that type argument especially!
- Note that it is a bit of a workaround for now as currently alex-c-line expects a JavaScript config file, so the TypeScript config must be built first.
- Eventually I will look into TypeScript configs more and see if we can allow for the TypeScript file to be run directly, or at least support building the file to JavaScript with `alex-c-line` natively. Keep an eye on the release notes for more information.
