This is a new major release of the `alex-c-line` package. It has the potential to introduce breaking changes that may require a large amount of refactoring. Please read the description of changes and migration notes below for more information.
<!-- alex-c-line-end-release-summary -->

## Description of Changes

- Refactor to use the subcommand pattern where appropriate.
    - See the first point of the migration notes for more information.
- Allow the `package-json` command to take a comma-separated list of rules under the `--rules` flag
    - As of now, we support doing `alex-c-line package-json --rules no-pre-release-dependencies,no-file-dependencies`
    - More rules will be added in the future, alongside config support to allow you to specify the exact rules you would like to enable.

## Migration Notes

- Change any scripts relying on the following commands:
    - `alex-c-line cache-path` -> `alex-c-line cache path`
    - `alex-c-line check-for-file-dependencies` -> `alex-c-line package-json --rules no-file-dependencies`
    - `alex-c-line package-json no-pre-release-dependencies` -> `alex-c-line package-json --rules no-pre-release-dependencies`
    - `alex-c-line update --check` -> `alex-c-line update check`
- The following commands have been moved to the `internal` subcommand group, meaning that it is not recommended for usage in production anymore:
    - `alex-c-line git-post-merge-cleanup`
    - `alex-c-line check-lockfile-version-discrepancy`
