This is a new major release of the `alex-c-line` package. It has the potential to introduce breaking changes that may require a large amount of refactoring. Please read the description of changes and migration notes below for more information.

## Description of Changes

<!-- user-editable-section-start -->
### Subcommand Pattern

- Refactor to use the subcommand pattern where appropriate.
    - Most flat commands have been grouped under structured subcommand namespaces (e.g. `version`, `template release-note`, `package-json`). Please update your scripts according to the migration notes.
- Allow the `package-json` command to take a comma-separated list of rules under the `--rules` flag
    - As of now, we support doing `alex-c-line package-json --rules no-pre-release-dependencies,no-file-dependencies`
    - More rules will be added in the future, alongside config support to allow you to specify the exact rules you would like to enable.

### Release Note Changes

- Add a `user-editable-section-<start|end>` markdown comment to the user editable section.
    - This not only makes it clearer for the users editing the document, but it also plays better with our automation.
- Change the existing `alex-c-line-<start|end>-release-status` and `alex-c-line-<start|end>-release-summary` comments to instead be `alex-c-line-release-status-<start|end>` and `alex-c-line-release-summary-<start|end>`
    - Putting the `start|end` at the end of the comment plays better with a newly added internal helper (`createMarkdownCommentPair`), and therefore our automation as a whole.
- None of the above changes requires you to refactor previous release notes, although future release notes (and any currently in progress) must be re-generated to make use of the new markdown comment headers.

### Pull Request Template Changes

- Add an `internal_change` pull request category to the `general` pull request template group.
    - This one is for changes to internal features that are technically accessible if you know where they are, but are not recommended for public usage.
    - Examples include: 
        - The `internal` subcommand from `alex-c-line`.
        - The `@alextheman/utility/internal` entrypoint from utility.
        - The `internal` config group in ESLint plugin.

### `local-package use` (previously `use-local-package`)

- Log information about the prepare script before executing it.
- Remove the experimental header - it is a usable command now and is no longer experimental.

### General Changes

- Change program errors to be exit code 2 instead of exit code 1 where appropriate.
    - Exit code 1 will now be for more generic errors (e.g. thrown by `DataError` or even regular `Error`), whereas exit code 2 represents more logical domain errors. (e.g. release note checking failures)
    - This separation is better for automation and makes it clearer exactly which errors are expected as a result of the command, and which are unexpected due to configuration issues etc.

## Migration Notes

- Change any scripts relying on the following commands:

### Package.json checks

- `alex-c-line check-for-file-dependencies` -> `alex-c-line package-json --rules no-file-dependencies`
- `alex-c-line package-json no-pre-release-dependencies` -> `alex-c-line package-json --rules no-pre-release-dependencies`

### Versioning

- `alex-c-line increment-version` -> `alex-c-line version increment`
- `alex-c-line get-version-type` -> `alex-c-line version type`
- `alex-c-line get-major-version` -> `alex-c-line version format --no-minor`
- `alex-c-line get-minor-version` -> `alex-c-line version format --no-patch`

### Release notes

- `alex-c-line create-release-note-2` -> `alex-c-line template release-note create`
- `alex-c-line check-release-note` -> `alex-c-line template release-note check`
- `alex-c-line migrate-release-notes` -> `alex-c-line template release-note migrate`[^1]
- `alex-c-line set-release-status-2` -> `alex-c-line template release-note set-status`

### Pull Request Templates

- `alex-c-line create-pull-request-template` -> `alex-c-line template pull-request create`

[^1] Will be removed as part of a minor release after this initial v2 release. Will be kept in initial v2 to give people a chance to migrate the release note directory structure.

### General

- `alex-c-line cache-path` -> `alex-c-line cache path`
- `alex-c-line update --check` -> `alex-c-line update check`
- `alex-c-line edit-env-file` -> `alex-c-line env-file edit`
- `alex-c-line pre-commit-2` -> `alex-c-line pre-commit`
- `alex-c-line use-local-package` -> `alex-c-line local-package use`

## Internal Commands

- The following commands have been moved to the `internal` subcommand group, meaning that it is not recommended for usage in production anymore:
    - `alex-c-line git-post-merge-cleanup`
    - `alex-c-line check-lockfile-version-discrepancy`

## Removals

- The legacy `pre-commit` implementation has been removed. The newer `pre-commit-2` implementation is now the canonical `pre-commit` command.
