This is a new minor release of the `alex-c-line` package. It introduces new features and other backwards-compatible changes that should require little to no refactoring. Please read the description of changes below.
<!-- alex-c-line-release-summary-end -->

## Description of Changes

<!-- user-editable-section-start -->
- Add an `internal dependency-audit` command.
  - This is an early proof-of-concept for a new command that creates a markdown file with a report about dependency health/security.
  - License checking will be added soon, but as of now the command includes results from:
    - `pnpm audit` for security auditing.
    - `pnpm peers check` for peer dependency issues.
    - `pnpm outdated` to check if there are any dependencies that are outdated.

## Additional Notes

- This is an early implementation as of now. Once I have developed this command further I do plan to eventually move this to the public command set.
- I think, at the very least it will support `pnpm` and `npm`, and in my case `pdm` will also be a good one to support next (though it may be its own separate command in case the audit commands themselves may differ slightly).
- I don't know if there any analogous tools/commands for GitHub Actions and Terraform, but if so, we could add those too. If anyone knows any, please let me know!
